# Geometry Dash Level Design Guide

## Основные правила создания уровней

### 1. Координаты и размещение

- **Ground level (земля)**: Y = 105-165 (обычно 165)
- **Единица измерения**: 30 пикселей = 1 блок
- **Расстояние между объектами**: минимум 30-60 единиц по X для комфортного прохождения
- **Высота прыжка**: ~150-200 единиц (5-7 блоков)

### 2. Базовые объекты

#### Блоки (ID: 1)
- Основа платформ
- Размещать на Y=165 для земли
- Можно ставить в ряд для длинных платформ

#### Шипы (ID: 2, 7, 8, 39)
- ID 2, 7: шип вверх (ставить на Y=165)
- ID 8: шип вниз (ставить на Y=300+)
- ID 39: шип вниз
- Расстояние между шипами: 60-90 единиц

#### Платформы (ID: 40)
- Для создания препятствий в воздухе
- Обычно Y=250-400

### 3. Порталы (обязательны для смены режима)

- **Cube Portal (ID: 12)**: стандартный режим
- **Ship Portal (ID: 13)**: полёт (управление удержанием)
- **Ball Portal (ID: 47)**: шар (переворот гравитации)
- **UFO Portal (ID: 111)**: НЛО (короткие прыжки)
- **Wave Portal (ID: 660)**: волна (свободный полёт)
- **Robot Portal (ID: 745)**: робот (высокие прыжки)
- **Spider Portal (ID: 1331)**: паук (телепорт вверх/вниз)

### 4. Орбы (для прыжков)

- **Yellow Orb (ID: 36)**: стандартный прыжок
- **Pink Orb (ID: 84)**: маленький прыжок
- **Red Orb (ID: 141)**: средний прыжок
- **Blue Orb (ID: 1022)**: высокий прыжок (гравитация)
- **Green Orb (ID: 1333)**: средний прыжок
- **Black Orb (ID: 1704)**: большой прыжок вниз

Размещение орбов:
- Над препятствиями на высоте 200-300
- Перед большими прыжками

### 5. Структура уровня

#### Начало уровня (X: 0-500)
1. Несколько блоков для разгона (X: 100-300, Y: 165)
2. Простые препятствия (1-2 шипа)
3. Первый прыжок с орбом

#### Средняя часть (X: 500-2000)
1. Комбинации блоков и шипов
2. Платформы в воздухе
3. Орбы для навигации
4. Смена режима (портал)

#### Конец уровня (X: 2000+)
1. Финальное испытание
2. Безопасная зона для приземления

### 6. Правила проходимости

✅ **ОБЯЗАТЕЛЬНО:**
- Земля (блоки) на Y=165 в начале
- Достаточно места для прыжков (150-200 единиц высоты)
- Орбы перед большими прыжками
- Безопасные зоны между сложными секциями

❌ **ИЗБЕГАТЬ:**
- Шипы вплотную к блокам (невозможно пройти)
- Слишком высокие прыжки без орбов (>250 единиц)
- Длинные пустые участки (скучно)
- Препятствия сразу после портала (нет времени среагировать)

### 7. Примеры секций

#### Простая платформа с прыжком:
```
X: 100-200, Y: 165 - блоки (земля)
X: 250, Y: 165 - шип
X: 300-400, Y: 165 - блоки (земля)
```

#### Прыжок с орбом:
```
X: 100-200, Y: 165 - блоки
X: 300, Y: 250 - yellow orb
X: 400-500, Y: 165 - блоки
```

#### Смена режима:
```
X: 500, Y: 200 - ship portal
X: 600-800, Y: 300 - платформы (для ship режима)
X: 900, Y: 200 - cube portal (возврат)
```

### 8. Триггеры для эффектов

#### Move Trigger
- Для движущихся платформ
- Ставить ПЕРЕД объектами (X - 100)
- Duration: 1-3 секунды для плавности

#### Alpha Trigger
- Для появления/исчезновения объектов
- Opacity: 0.0 (невидимый) - 1.0 (видимый)

#### Rotate Trigger
- Для вращающихся препятствий
- 360 градусов = полный оборот

### 9. Группы объектов

- Группа 1: основные движущиеся платформы
- Группа 2: декорации
- Группа 3: враги/препятствия
- Группа 4+: специальные эффекты

### 10. Цветовые каналы

- 0: цвет по умолчанию
- 1-5: основные цвета уровня
- Использовать для визуального разделения секций

## Пример простого уровня

```python
# Начало - разгон
create_object(1, 100, 165, "0", 0)  # блок
create_object(1, 150, 165, "0", 0)  # блок
create_object(1, 200, 165, "0", 0)  # блок

# Первое препятствие
create_object(2, 300, 165, "0", 0)  # шип

# Платформа после шипа
create_object(1, 400, 165, "0", 0)  # блок
create_object(1, 450, 165, "0", 0)  # блок

# Прыжок с орбом
create_object(36, 550, 250, "0", 0)  # yellow orb

# Приземление
create_object(1, 650, 165, "0", 0)  # блок
create_object(1, 700, 165, "0", 0)  # блок
```

## Советы для AI

1. **Всегда начинайте с земли** (блоки на Y=165)
2. **Размещайте триггеры ПЕРЕД объектами** (X - 50-100)
3. **Оставляйте пространство** между препятствиями (60-90 единиц)
4. **Тестируйте логику**: может ли игрок физически пройти?
5. **Используйте орбы** для сложных прыжков
6. **Добавляйте порталы** для разнообразия геймплея
